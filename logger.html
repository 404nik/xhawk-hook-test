Telemetry Logger

Structured JSON logging for xHawk Hook Test

Overview

The logger provides structured JSON output with ISO timestamps, log levels, and process metadata. It's designed for tracing telemetry data in Next.js applications and can also serve as middleware for HTTP request logging.

Log Levels

LevelValueUse Case
DEBUG0Verbose development output
INFO1General application events
WARN2Potential issues or deprecations
ERROR3Failures and exceptions

Basic Usage

const logger = require('./logger');

// Simple logging
logger.info("Server started", { port: 3000 });
logger.warn("Slow query", { durationMs: 1200 });
logger.error("Connection failed", { host: "db.internal" });

Telemetry Events

Use the dedicated telemetry() method to log structured telemetry data with event names and metadata.

// Track telemetry events
logger.telemetry("page_view", {
  path: "/dashboard",
  userId: "user_abc",
  sessionId: "sess_123"
});

logger.telemetry("commit_traced", {
  commitHash: "078e1cf",
  branch: "test/session-traces-v2"
});

Request Middleware

Attach requestLogger as middleware to log every incoming request and its response time.

const { requestLogger } = require('./logger');

// Express / Next.js custom server
app.use(requestLogger);
